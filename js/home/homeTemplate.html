Nex is alpha software. It's an experiment to find simple and intuitive UI design patterns. The RequireJS Router is production ready. It's a thin layer on top of RequireJS to provide lazy loading routing.

{nex.js}

Semantic, declarative, AMD web apps.

Everything is an element! Nex components are like views but they're built on plain DOM elements. The component's API is exposed on the DOM element!

Nex is declarative. Events are defined in the template. So are sub-components. You know exactly what the component is doing by looking at the template. Plus, you can use any template engine you want!

Components are encapsulated. Every component is rendered before being attached to the document or other components. Events only trigger event handlers in their own component.

Why Nex

First, get to know AMD. AMD is the future of JavaScript dependency injection. Modules clearly define their dependencies. require.js is the only script tag you include. The rest of your scripts are loaded using AMD. No more globals!

Nex components and the RequireJS Router give you a powerful IoC pattern for routing and loading views. The router lazy loads components as you navigate to each page. Nex renders the component, its layout, and any sub-components. The layout is now completely decoupled from the router.

The combination of AMD, a lazy loading router, and Nex components gives you an awesome toolset to build your application. This is the next step in web application design.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Components

homePage.html and homePage.js

Route directly to your home page.

The home page renders its layout and components.

Events are declared in the template and are scoped to this component.

<h1>Hello \{{model.name}}!</h1>
<input type="text" on-keypress="updateNameOnEnter">
<component name="todo-app"></component>
var HomePage = Nex.defineComponent('home-page', {
  template: Handlebars.compile(homePageTemplate),
  layout: Layout,
  model: { name: 'Nex' },
  updateNameOnEnter: function(event) {
    if (event.keyCode === 13) {
      this.model.name = event.target.value;
      this.render();
    }
  },
  components: [TodoApp] // Registers the todo-app component
});

layout.html and layout.js

The layout contains your site's header, footer, etc.

HomePage will attach to content-placeholder.

<header>
  <a href="#/" \{{#if model.home}}class="active"\{{/if}}>Home</a>
  <a href="#/examples" \{{#if model.examples}}class="active"\{{/if}}>Examples</a>
</header>
<!-- Every layout must contain a content-placeholder -->
<component name="content-placeholder"></component>
var Layout = Nex.defineComponent('layout', {
  template: Handlebars.compile(layoutTemplate),
  model: { home: true, examples: false }
});

todoApp.html and todoApp.js

TodoApp defines the todo-app component. This is automatically rendered and attached to the HomePage.

<h2>Todo MVC</h2>
var TodoApp = Nex.defineComponent('todo-app', {
  template: Handlebars.compile(todoAppTemplate)
});

Calling var homePage = new HomePage() will render the home page, its layout, and components.

Attach it to the document using homePage.attachTo('body'). This will attach homePage.outerEl which includes the layout.

<component name="layout"> <!-- layout, homePage.outerEl -->
  <header>
    <a href="#/" class="active">Home</a>
    <a href="#/examples">Examples</a>
  </header>
  <component name="home-page"> <!-- home-page replaced the content-placeholder -->
    <h1>Hello Nex!</h1>
    <input type="text" on-keypress="updateNameOnEnter">
    <component name="todo-app">
      <h2>Todo MVC</h2>
    </component>
  </component>
