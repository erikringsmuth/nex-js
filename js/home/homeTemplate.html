{nex.js}

Powerful, modular, AMD compatible views.

The combination of nex-js views and the RequireJS Router gives you a simple, powerful, modular framework to build your next web app. The AMD router lazy loads dependencies. Nex gives you layout views which make routing and rendering easy. It's fully AMD compatible plus it's built on vanilla JS, no jQuery required.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Why Nex

First, why AMD. AMD is the future of Javascript dependency injection. The only script tag you should ever include should load require.js. The rest of your scripts should be loaded by require.js and injected into your modules.

The router comes next. Other frameworks load all of your app's Javascript up front. You can optimize your site by compiling it down to one file but that still doesn't solve the problem. The RequireJS Router lazy loads your modules when you navigate to the route. You're site could contain 10mb of Javascript and HTML templates and it would only load the 100kb needed for the current page.

Last, next-js views. Once you're using AMD and a lazy loading router you can route to your views. Normal view's can't specify a layout (header, footer, etc.). You have to render the layout then use the router to load the content. With Nex Views you route straight to your view and the view specifies a layout that includes your header and footer. Rendering your view also renders the layout and you attach your view.outerEl to the document with everything rendered.

The combination of AMD, a lazy loading router, and layout views gives you a simple, powerful, modular framework to build your application. This is the next step in web applications.

Browser Support

Browser support is limited to browsers that support document.querySelector() and its variants.
