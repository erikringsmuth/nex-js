Best Practices

Avoid globals with AMD. This is what require.js is for. require.js is the only script tag you should ever include. The rest of your scripts should be injected into your modules using require.js.

'use strict'; to enforce safer Javascript practices. [MDN Strict Mode]

If your overriding a function like render() it's helpful to return this; at the end.

Nex.defineComponent('my-component', {
  render: function render() {
    this.html(this.template({model: this.model}));
    return this;
  }
});

this refers to the instance of your component. Now you can write chained calls like new MyComponent().render().outerEl. If render() didn't return this then render().outerEl would throw TypeError: Cannot read property 'outerEl' of undefined.

The reason why the function name is repeated when declaring a function like render: function render() { ... is so that the name of the function will show up in a stack trace. If you declare a function like render: function() { ... it will show up as an anonymous function.
