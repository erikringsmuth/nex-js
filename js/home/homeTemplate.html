Nex is alpha software. It's an experiment to find simple and intuitive UI design patterns. The RequireJS Router is production ready. It's a thin layer on top of RequireJS to provide simple lazy loading routing.

{nex.js}

Semantic, declarative, AMD web apps.

Everything is an element! Nex components are like views but they're built on plain DOM elements. The component's API is on the DOM element!

Nex.js and the RequireJS Router give you a powerful IoC pattern for routing and loading views. The router lazy loads component. Nex renders the component and injects the layout. The layout is now completely decoupled from the router. It's fully AMD compatible plus it's built on vanilla JS.

Why Nex

AMD is the future of Javascript dependency injection. Modules clearly define their dependencies. require.js is the only script tag you include. The rest of your scripts are loaded using AMD. No more globals!

The RequireJS Router lazy loads modules as you navigate to each page. You're site could contain 10MB of Javascript and HTML templates and it would only load the 10KB for the current page. With Nex the router loads your component and your component injects the layout. You attach component.outerEl to the document with everything rendered.

The combination of AMD, a lazy loading router, and layouts gives you an awesome toolset to build your application. This is the next step in web application design.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Components

homePage.html and homePage.js

Route directly to your home page. The home page has a layout and components. Events are declared in the template and are scoped to this component.

<h1>Hello \{{model.name}}!</h1>
<input type="text" on-keypress="updateNameOnEnter">
<component name="todo-app"></component>
var HomePage = Nex.defineComponent('home-page', {
  template: Handlebars.compile(homePageTemplate),
  layout: Layout,
  model: { name: 'Nex' },
  updateNameOnEnter: function(event) {
    if (event.keyCode === 13) {
      this.model.name = event.target.value;
      this.render();
    }
  },
  components: [TodoApp]
});

layout.html and layout.js

The layout contains your site's header, footer, etc. The HomePage will be attached to the content-placeholder.

<header>
  <a href="#/" \{{#if model.home}}class="active"\{{/if}}>Home</a>
  <a href="#/examples" \{{#if model.examples}}class="active"\{{/if}}>Examples</a>
</header>
<component name="content-placeholder"></component>
var Layout = Nex.defineComponent('layout', {
  template: Handlebars.compile(layoutTemplate),
  model: { home: true, examples: false }
});

todoApp.html and todoApp.js

The home page will inject this TodoMVC app.

<h2>Todo MVC</h2>
var TodoApp = Nex.defineComponent('todo-app', {
  template: Handlebars.compile(todoAppTemplate)
});

Calling var homePage = new HomePage() will render the home page, its layout, and components. Attach it to the document using homePage.attachTo('body'). This will attach homePage.outerEl which includes the layout.

<component name="layout"> <!-- layout, homePage.outerEl -->
  <header>
    <a href="#/" class="active">Home</a>
    <a href="#/examples">Examples</a>
  </header>
  <!-- the home page injected itself into the layout's `content-placeholder` component -->
  <component name="home-page">
    <h1>Hello Nex!</h1>
    <input type="text" on-keypress="updateNameOnEnter">
    <component id="todo-app">
      <h2>Todo MVC</h2>
    </component>
  </component>
