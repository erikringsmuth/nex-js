{nex.js}

Unleashing the power of AMD for web applications.

Nex.js and the RequireJS Router give you a powerful IoC pattern for routing and loading views. The router lazy loads views. Nex renders the view and it's layout. Rendering the layout is now completely decoupled from the router. It's fully AMD compatible plus it's built on vanilla JS.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Why Nex

First, why AMD. AMD is the future of Javascript dependency injection. require.js is the only script tag you need to include. The rest of your scripts are injected using AMD. No more globals!

The router comes next. Other frameworks load all of your app's Javascript up front. You can optimize your site by compiling it down to one file but that still doesn't solve the problem. The RequireJS Router lazy loads modules as you navigate to each page. You're site could contain 10MB of Javascript and HTML templates and it would only load the 10KB needed for the current page.

Last, nex-js views. Other frameworks render the layout (header, footer, etc.) then use the router to load the content. With nex-js the router loads your view and your view injects the layout. You attach your view.outerEl to the document with everything rendered.

The combination of AMD, a lazy loading router, and layout views gives you an awesome framework to build your application. This is the next step in web application design.

Layout Views

var LayoutView = Nex.View.extend({
  template: Handlebars.compile(layoutTemplate),
  model: { home: true, examples: false },
  contentPlaceholder: '#content'
});
<header>
  <a href="#/" \{{#if model.home}}class="active"\{{/if}}>Home</a>
  <a href="#/examples" \{{#if model.examples}}class="active"\{{/if}}>Examples</a>
</header>
<main id="content"></main>

LayoutView and layoutTemplate.html

The layout contains your site's header, footer, etc. This will get rendered on every page.

var HomeView = Nex.View.extend({
  template: Handlebars.compile(homeTemplate),
  layout: LayoutView
});
<h1>Hello World!</h1>

HomeView and homeTemplate.html

Route directly to your home view. The home view specifies a layout.

<div> <!-- layoutView.el, homeView.outerEl -->
  <header>
    <a href="#/" class="active">Home</a>
    <a href="#/examples">Examples</a>
  </header>
  <main id="content">
    <div> <!-- homeView.el -->
      <h1>Hello World!</h1>
    </div>
  </main>
<div>

Creating a new HomeView() will render the home view and layout.
