{nex.js}

Powerful, modular, AMD compatible views.

Nex views and the RequireJS Router give you a simple, powerful, modular framework to build your next web app. The AMD router lazy loads dependencies. Nex gives you layout views. It's fully AMD compatible plus it's built on vanilla JS, no jQuery required.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Why Nex

First, why AMD. AMD is the future of modular Javascript and dependency injection. require.js is the only script tag you should ever include. The rest of your scripts should be injected into your modules using require.js. No more globals!

The router comes next. Other frameworks load all of your app's Javascript up front. You can optimize your site by compiling it down to one file but that still doesn't solve the problem. The RequireJS Router lazy loads your modules as you navigate to the route. You're site could contain 10mb of Javascript and HTML templates and it would only load the 20kb needed for the current page.

Last, nex-js views. Other frameworks render the layout (header, footer, etc.) then use the router to load the content. With nex-js the router loads your view and your view injects the layout. You attach your view.outerEl to the document with everything rendered.

The combination of AMD, a lazy loading router, and layout views gives you an awesome framework to build your application. This is the next step in web application design.

Browser Support

Browser support is limited by document.querySelector() and its variants. IE8 has additional problems with element.innerHTML.
