Nex is alpha software. It's an experiment to find simple and intuitive UI design patterns. The RequireJS Router is production ready. It's a thin layer on top of RequireJS to provide simple lazy loading routing.

{nex.js}

Unleashing the power of AMD for web applications.

Nex.js and the RequireJS Router give you a powerful IoC pattern for routing and loading views. The router lazy loads views. Nex renders the view and injects the layout. The layout is now completely decoupled from the router. It's fully AMD compatible plus it's built on vanilla JS.

Check out the examples to see how it works. This site's source is a great example of a nex-js and RequireJS Router setup. It's available in the gh-pages branch of nex-js. Licensed under MIT.

Why Nex

First, why AMD. AMD is the future of Javascript dependency injection. Modules clearly define their dependencies. require.js is the only script tag you include. The rest of your scripts are loaded using AMD. No more globals!

The router comes next. Other frameworks load all of your app's Javascript up front. You can optimize your site by compiling it down to one file but that doesn't scale. The RequireJS Router lazy loads modules as you navigate to each page. You're site could contain 10MB of Javascript and HTML templates and it would only load the 10KB for the current page.

Last, nex-js views. Other frameworks render the layout (header, footer, etc.) then use the router to load the content. With nex-js the router loads your view and your view injects the layout. You attach view.outerEl to the document with everything rendered.

The combination of AMD, a lazy loading router, and layout views gives you an awesome toolset to build your application. This is the next step in web application design.

Components

homePage.html and homePage.js

Route directly to your home page. The home page has a layout and components. Events are declared in the template and are scoped to this view.

<h1>Hello \{{model.name}}!</h1>
<input type="text" on-keypress="updateNameOnEnter">
<component name="todo-app"></component>
var HomePage = Nex.defineComponent('home-page', {
  template: Handlebars.compile(homePageTemplate),
  layout: Layout,
  model: { name: 'Nex' },
  updateNameOnEnter: function(event) {
    if (event.keyCode === 13) {
      this.model.name = event.target.value;
      this.render();
    }
  },
  components: [TodoApp]
});

layout.html and layout.js

The layout contains your site's header, footer, etc. This will get rendered on every page that has this layout.

<header>
  <a href="#/" \{{#if model.home}}class="active"\{{/if}}>Home</a>
  <a href="#/examples" \{{#if model.examples}}class="active"\{{/if}}>Examples</a>
</header>
<component name="content-placeholder"></component>
var Layout = Nex.defineComponent('layout', {
  template: Handlebars.compile(layoutTemplate),
  model: { home: true, examples: false }
});

todoApp.html and todoApp.js

The TodoMVC app is injected into the home page.

<h2>Todo MVC</h2>
var TodoApp = Nex.defineComponent('todo-app', {
  template: Handlebars.compile(todoAppTemplate)
});

var homePage = new HomePage() will render the home page, its layout, and components. Attach it to the document using homePage.attachTo('body'). This will attach homePage.outerEl which includes the layout.
